## 2. 시스템 설계 문서 (System Prompt Engineering)

### 1) 시스템 제약 조건 및 대응 전략
* **플랫폼 하드 제약:** '학습 네이토' 내부의 상위 지침(AI 학습 튜터 페르소나)이 강제되어 있어, 완전한 비즈니스 톤으로의 페르소나 전환 및 출력 서식 100% 강제는 시스템적으로 거부됨을 확인함.
* **우회 설계 전략:** AI의 페르소나 자체를 개조하는 대신, AI가 제안한 타협안을 수용하여 요약 정보의 '알맹이(핵심 데이터 구조)'를 정확히 출력하도록 프롬프트를 조율함.

### 2) 주입된 최종 프롬프트 구조
* **입력 데이터 구조화:** 회의록 원문 분리 제공
* **조건 누적 반영:** 사내 인적 자산(원고 담당 '형', 검수 담당 '이모부') 및 예산 한도(200만 원 초과 시 이모부 사전 승인 필수) 조건을 명시적으로 누적 주입.
